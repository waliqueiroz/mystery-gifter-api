## Key Design Decisions

### 1. `Group.IsMember` + `Group.CanView` (BC-003)

`IsMember(userID string) bool` iterates `group.Users` and returns true if found. `CanView(requesterID string) error` calls `IsMember` and returns `NewForbiddenError` on failure. The `GroupService.GetByID` signature changes to `GetByID(ctx, groupID, requesterID string)`.

All callers of `groupService.GetByID` in controllers and services must be updated:
- `GroupController.GetByID` — passes `authUserID`
- `GroupInviteService.JoinGroup` — calls `groupRepository.GetByID` directly (not through service), so unaffected

### 2. `GroupInviteService.GetActive` (BC-002)

Checks group membership before returning the invite. Flow:
1. `groupRepository.GetByID(ctx, groupID)` → 404 if group not found
2. `group.CanView(requesterID)` → 403 if not a member
3. `groupInviteRepository.GetActiveByGroupID(ctx, groupID)` → 404 if no active invite

### 3. `GroupDTO` without `Matches` (BC-004)

`mapGroupFromDomain` no longer calls `mapMatchesFromDomain`. The `match_dto.go` mapper is still present (not dead code — may be used by future endpoints or tests). Only the `GroupDTO` struct and its mapper are changed.

### 4. No migration required (user note)

The system is not in production. Existing migrations may be edited if needed. For this feature, no migration is needed at all — the `group_invites` table already has the necessary columns.
